## 第一部分: 概述

为项目添加错误边界（Error Boundary）和加载状态（Loading Skeleton）支持。创建了 ErrorBoundary 组件捕获组件树的 JavaScript 错误，创建了各种 Skeleton 组件用于页面加载时的占位显示，并添加了 app 级别的 error.tsx 处理路由级错误。Skeleton 组件已集成到首页、用户资料页、文章详情页和写文章页。

## 第三部分: 详细改动说明

### error-boundary.tsx[新增](apps/web/src/components/error-boundary.tsx)

```typescript
export class ErrorBoundary extends Component<Props, State> {
  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error };
  }

  handleRetry = () => {
    this.setState({ hasError: false, error: null });
    window.location.reload();
  };

  render() {
    if (this.state.hasError) {
      return (
        <div className="flex flex-col items-center justify-center min-h-[400px]">
          <h2>出错了</h2>
          <p>抱歉，页面加载时遇到了问题。请尝试刷新页面。</p>
          <Button onClick={this.handleRetry}>刷新页面</Button>
        </div>
      );
    }
    return this.props.children;
  }
}
```

**说明：**
- 使用 React Class Component 实现 Error Boundary（React 要求）
- `getDerivedStateFromError` 在错误时更新状态并渲染错误 UI
- `handleRetry` 调用 `window.location.reload()` 刷新页面
- 开发模式下显示详细错误信息

### skeleton.tsx[新增](apps/web/src/components/loading/skeleton.tsx)

```typescript
export function Skeleton({ className, 'data-testid': testId }) {
  return (
    <div data-testid={testId || 'skeleton'} className={`animate-pulse rounded-md bg-muted ${className || ''}`} />
  );
}

export function ArticleCardSkeleton() {
  return (
    <div data-testid="article-card-skeleton" className="border-b border-border p-4">
      <div className="flex gap-4">
        <Skeleton className="h-20 w-20 rounded" />
        <div className="flex-1 space-y-2">
          <Skeleton className="h-4 w-3/4" />
          <Skeleton className="h-4 w-1/2" />
        </div>
      </div>
    </div>
  );
}

export function ArticleListSkeleton({ count = 5 }) {
  return (
    <div>
      {Array.from({ length: count }).map((_, i) => (
        <ArticleCardSkeleton key={i} />
      ))}
    </div>
  );
}

export function ProfileHeaderSkeleton() {
  return (
    <div data-testid="profile-header-skeleton" className="bg-background border-b p-8">
      <div className="flex items-start gap-6">
        <Skeleton className="h-20 w-20 rounded-full" />
        <div className="flex-1 space-y-3">
          <Skeleton className="h-6 w-32" />
          <Skeleton className="h-4 w-20" />
          <Skeleton className="h-8 w-24" />
        </div>
      </div>
    </div>
  );
}

export function ArticleEditorSkeleton() {
  return (
    <div data-testid="article-editor-skeleton" className="max-w-3xl mx-auto">
      <div className="border rounded-md p-6 space-y-4">
        <Skeleton className="h-10 w-full" />
        <Skeleton className="h-10 w-full" />
        <Skeleton className="h-96 w-full" />
        <Skeleton className="h-24 w-full" />
        <Skeleton className="h-10 w-1/3" />
      </div>
    </div>
  );
}
```

**说明：**
- `Skeleton` - 基础骨架组件，使用 `animate-pulse` 实现闪烁效果
- `ArticleCardSkeleton` - 文章卡片骨架屏
- `ArticleListSkeleton` - 文章列表骨架屏，支持自定义数量
- `ProfileHeaderSkeleton` - 用户资料头部骨架屏
- `ArticleEditorSkeleton` - 文章编辑器骨架屏

## 其它

### 设计亮点

- **Error Boundary 模式**：使用 React 原生 Error Boundary API，能捕获子树中的 JS 错误
- **Skeleton 动画**：使用 Tailwind `animate-pulse` 实现流畅的加载动画
- **路由级错误处理**：Next.js App Router 的 error.tsx 自动处理整个路由组的错误
- **统一的 Skeleton**：集中管理所有骨架屏组件，页面复用简单

### 使用建议

在以下场景使用骨架屏：
- 文章列表页加载时 → `ArticleListSkeleton`
- 用户资料页加载时 → `ProfileHeaderSkeleton` + `ArticleListSkeleton`
- 文章详情页加载时 → `ArticleEditorSkeleton`
- 文章编辑器加载时 → `ArticleEditorSkeleton`
