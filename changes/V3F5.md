### 3.4 Drag and Drop Support

**File:** `apps/web/src/components/tiptap/tiptap-editor.tsx`

**新增 editorProps.handleDrop 配置：**

```typescript
editorProps: {
  handleDrop: (view, event, _slice, _moved) => {
    const files = event.dataTransfer?.files;
    if (!files) return false;

    const file = files[0];
    if (!isImageFile(file)) return false;
    if (file.size > getImageFileLimit()) {
      alert('图片大小不能超过 5MB');
      return true;
    }

    // 阻止默认行为，读取文件并上传
    event.preventDefault();
    const reader = new FileReader();
    reader.onload = async () => {
      const result = await uploadImage(file);
      if (result.success && result.url) {
        editor?.chain().focus().setImage({ src: result.url }).run();
      }
    };
    reader.readAsDataURL(file);
    return true;
  },
}
```

## 5. User Flow

### 点击上传
1. 用户点击工具栏的 🖼 按钮
2. 弹出文件选择对话框
3. 选择图片文件
4. 显示上传中状态
5. 上传成功后图片插入编辑器

### 拖拽上传
1. 用户从桌面/文件夹拖拽图片到编辑器
2. 编辑器检测到图片文件
3. 自动读取并上传到服务器
4. 上传成功后图片插入到拖拽位置

### URL 输入
1. 用户点击工具栏的 🔍 按钮
2. 弹出输入框要求输入图片 URL
3. 输入 URL 后图片直接插入编辑器

## 7. API Specification

### POST /api/upload

**Request:**
```
Content-Type: multipart/form-data
Body: { file: <image_file> }
```

**Response (Success):**
```json
{
  "success": true,
  "url": "https://cdn.example.com/images/abc123.jpg"
}
```

**Response (Error):**
```json
{
  "success": false,
  "error": "File size exceeds 5MB limit"
}
```

## 9. Manual Testing Guide

### 前提条件
1. 启动后端服务（确保 `http://localhost:4000` 可用）
2. 启动前端开发服务器 `pnpm --filter @jianshu/web dev`
3. 登录一个已注册账号

### 测试 1: 图片 URL 插入

**步骤：**
1. 访问 `/write` 进入文章编辑页面
2. 在富文本编辑器中点击工具栏的 🔍 按钮（图片 URL）
3. 在弹窗中输入任意图片 URL，例如：`https://picsum.photos/400/300`
4. 点击确定
5. **预期结果：** 图片显示在编辑器中

### 测试 2: 图片文件上传

**步骤：**
1. 在编辑器中点击工具栏的 🖼 按钮（上传图片）
2. 在文件选择对话框中选择一张小于 5MB 的图片文件
3. **预期结果：**
   - 按钮显示 ⏳ 状态
   - 上传完成后图片插入到编辑器中

**错误场景测试：**
1. 选择一个大于 5MB 的文件
2. **预期结果：** 弹出提示 "图片大小不能超过 5MB"

### 测试 3: 拖拽上传

**步骤：**
1. 打开一个新的浏览器标签页，访问任意图片网站（如 `https://picsum.photos`）
2. 右键点击一张图片，选择「复制图片」
3. 回到编辑器，在编辑区域内右键选择「粘贴」
4. **预期结果：** 图片被上传并插入到编辑器中

**注意：** 如果粘贴不生效，可以尝试直接将图片文件从文件夹拖入编辑器区域

### 测试 4: 上传失败处理

**步骤：**
1. 确保后端服务已停止或 `/api/v1/upload` 不可用
2. 尝试上传图片
3. **预期结果：** 弹出提示 "上传失败"
