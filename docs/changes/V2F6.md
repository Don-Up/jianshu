## 1. Notification 类型定义

**文件:** `packages/shared/src/index.ts`

在 shared 包中添加 `Notification` 接口，与 Prisma Schema 中的 Notification 模型对应：

```typescript
export interface Notification {
  id: string;
  type: 'COMMENT' | 'LIKE' | 'FOLLOW' | 'SYSTEM';
  message: string;
  link?: string | null;
  isRead: boolean;
  createdAt: Date;
  actor?: Pick<User, 'id' | 'username' | 'name' | 'avatar'> | null;
  article?: { id: string; title: string; slug: string } | null;
}
```

**设计说明:**

- `type` 字段支持四种通知类型
- `actor` 使用 `Pick<User>` 避免循环依赖，只包含展示所需字段
- `article` 可选，关联的文章信息（用于跳转）
- `link` 可选，直接链接到目标页面

## 2. notificationApi

**文件:** `apps/web/src/lib/api.ts`

添加 `notificationApi`，提供三个方法：

```typescript
export const notificationApi = {
  list: (params?: { page?: number; limit?: number }) => {
    const searchParams = params
      ? new URLSearchParams(
          Object.entries(params)
            .filter(([, v]) => v !== undefined)
            .map(([k, v]) => [k, String(v)])
        ).toString()
      : '';
    return fetchApi<PaginatedResponse<Notification>>(
      `/api/notifications${searchParams ? `?${searchParams}` : ''}`
    );
  },

  markAsRead: (id: string) =>
    fetchApi<void>(`/api/notifications/${id}/read`, { method: 'POST' }),

  markAllAsRead: () =>
    fetchApi<void>('/api/notifications/read-all', { method: 'POST' }),
};
```

**API 对应:**

| 方法 | 端点 | 功能 |
|------|------|------|
| `list` | GET `/api/notifications` | 获取通知列表 |
| `markAsRead` | POST `/api/notifications/:id/read` | 标记单条已读 |
| `markAllAsRead` | POST `/api/notifications/read-all` | 全部标记已读 |

## 3. useNotifications Hook

**文件:** `apps/web/src/hooks/use-notifications.ts`

使用 TanStack Query 管理通知状态：

```typescript
export function useNotifications(): UseNotificationsResult {
  const queryClient = useQueryClient();

  const query = useQuery({
    queryKey: ['notifications'],
    queryFn: () => notificationApi.list(),
  });

  const markAsReadMutation = useMutation({
    mutationFn: (id: string) => notificationApi.markAsRead(id),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['notifications'] });
    },
  });

  const markAllAsReadMutation = useMutation({
    mutationFn: () => notificationApi.markAllAsRead(),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['notifications'] });
    },
  });

  const notifications = query.data?.data?.items || [];
  const total = query.data?.data?.total || 0;
  const unreadCount = notifications.filter((n) => !n.isRead).length;

  return {
    notifications,
    total,
    unreadCount,
    isLoading: query.isLoading,
    markAsRead: markAsReadMutation.mutate,
    markAllAsRead: markAllAsReadMutation.mutate,
  };
}
```

**返回值的意义:**

| 属性 | 类型 | 说明 |
|------|------|------|
| `notifications` | `Notification[]` | 通知列表 |
| `total` | `number` | 通知总数 |
| `unreadCount` | `number` | 未读数量 |
| `isLoading` | `boolean` | 加载状态 |
| `markAsRead` | `(id: string) => void` | 标记单条已读 |
| `markAllAsRead` | `() => void` | 全部标记已读 |

**设计说明:**

- 使用 `useMutation` 的 `onSuccess` 回调自动 invalidate 查询，确保数据一致性
- `unreadCount` 通过过滤 `isRead === false` 计算，而非单独请求 API

## 4. NotificationList 组件

**文件:** `apps/web/src/components/notifications/notification-list.tsx`

核心组件，负责渲染通知列表：

```typescript
export function NotificationList() {
  const { notifications, isLoading, markAsRead, markAllAsRead } = useNotifications();

  if (isLoading) {
    return (
      <div className="space-y-4">
        {[1, 2, 3].map((i) => (
          <div key={i} className="h-16 bg-muted rounded-lg animate-pulse" />
        ))}
      </div>
    );
  }

  if (notifications.length === 0) {
    return <p className="text-center text-muted-foreground py-8">暂无通知</p>;
  }

  return (
    <div>
      <div className="flex justify-between items-center mb-4">
        <h2 className="text-lg font-semibold">通知</h2>
        <button onClick={() => markAllAsRead()} className="text-sm text-primary hover:underline">
          全部标为已读
        </button>
      </div>
      <div className="space-y-2">
        {notifications.map((notification: Notification) => (
          <Link
            key={notification.id}
            href={notification.link || '#'}
            onClick={() => !notification.isRead && markAsRead(notification.id)}
            className={cn(
              'flex gap-3 p-4 rounded-lg transition-colors',
              notification.isRead ? 'bg-background' : 'bg-primary/5'
            )}
          >
            {/* Avatar, Content, Unread indicator */}
          </Link>
        ))}
      </div>
    </div>
  );
}
```

**交互设计:**

| 状态 | 样式 |
|------|------|
| 已读通知 | `bg-background`，白色背景 |
| 未读通知 | `bg-primary/5`，淡蓝色背景 |
| 未读指示器 | 右侧蓝色圆点 `bg-primary` |
| "全部标为已读" | `text-primary hover:underline` |

**点击行为:**

- 点击通知项 → 调用 `markAsRead(id)`（仅未读时）→ 跳转链接
- 点击"全部标为已读" → 调用 `markAllAsRead()`


```
       ↓

       ↓

       ↓

       ↓

       ↓
用户点击通知
       ↓
```
