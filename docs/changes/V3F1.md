## 详细改动

### 1. API 客户端 (`apps/web/src/lib/api.ts`)

新增两个 API 方法：

```typescript
export const userApi = {
  // ... 现有方法

  updateProfile: (data: { name?: string; bio?: string; avatar?: string }) =>
    fetchApi<User>('/api/users/me', {
      method: 'PATCH',
      body: JSON.stringify(data),
    }),

  changePassword: (data: { oldPassword: string; newPassword: string }) =>
    fetchApi<void>('/api/users/me/change-password', {
      method: 'POST',
      body: JSON.stringify(data),
    }),
};
```

**说明**:
- `updateProfile`: 调用 `PATCH /api/users/me` 更新用户资料
- `changePassword`: 调用 `POST /api/users/me/change-password` 修改密码

### 2. Auth Context (`apps/web/src/hooks/use-auth.tsx`)

添加 `updateUser` 方法，用于更新全局用户状态：

```typescript
interface AuthContextType {
  // ... 现有字段
  updateUser: (user: User) => void;
}

// 实现
const updateUser = useCallback((updatedUser: User) => {
  setUserState(updatedUser);
  saveUser(updatedUser);
}, []);
```

**说明**:
- 当用户在设置页面更新资料后，同步更新全局的用户状态
- 同时更新 localStorage 保持持久化

### 3. Settings Hook (`apps/web/src/hooks/use-settings.ts`)

新建 Hook，提供设置相关的状态管理：

```typescript
interface UseSettingsResult {
  updateProfile: (data: UpdateProfileData) => Promise<void>;
  changePassword: (data: ChangePasswordData) => Promise<void>;
  isUpdatingProfile: boolean;
  isChangingPassword: boolean;
  updateProfileError: string | null;
  changePasswordError: string | null;
}
```

**特性**:
- 使用 TanStack Query 的 `useMutation` 管理异步状态
- `updateProfile` 成功后自动 invalidate `['auth', 'me']` 查询
- 提供加载状态和错误信息

### 4. 设置页面 (`apps/web/src/app/(app)/settings/page.tsx`)

实现完整的设置页面，包含两个表单卡片：

#### 个人信息表单

- **头像 URL 输入框**: 可输入头像图片 URL
- **昵称输入框**: 必填，验证非空
- **个人简介输入框**: 可选
- **用户名**: 禁用，显示当前用户名
- **邮箱**: 禁用，显示当前邮箱
- **保存按钮**: 调用 `updateProfile` API

#### 密码修改表单

- **当前密码**: 必填
- **新密码**: 必填，至少 6 个字符
- **确认新密码**: 必填，需与新密码一致
- **修改按钮**: 调用 `changePassword` API

#### 表单验证

```typescript
// 个人信息验证
if (!name.trim()) {
  toast.error('昵称不能为空');
  return;
}

// 密码修改验证
if (newPassword.length < 6) {
  toast.error('新密码至少6个字符');
  return;
}
if (newPassword !== confirmPassword) {
  toast.error('两次输入的新密码不一致');
  return;
}
```

#### Toast 通知

使用 `sonner` 库显示成功/失败提示：

```typescript
toast.success('个人信息更新成功');
toast.error('更新失败，请重试');
toast.success('密码修改成功');
toast.error('密码修改失败，请检查当前密码是否正确');
```

## 用户体验流程

### 更新个人信息

1. 用户在设置页面修改昵称/简介/头像 URL
2. 点击"保存修改"按钮
3. 显示"保存中..."加载状态
4. 成功后显示 Toast 提示，全局用户状态同步更新
5. 失败显示错误提示

### 修改密码

1. 用户填写当前密码、新密码、确认密码
2. 点击"修改密码"按钮
3. 前端验证：非空、长度>=6、两次输入一致
4. 显示"修改中..."加载状态
5. 成功后清空表单，显示成功 Toast
6. 失败显示错误提示

## 设计亮点

1. **原生表单验证**: 无需额外库，使用条件判断实现简单验证
2. **乐观更新**: 表单提交后立即更新全局状态
3. **用户体验**: 加载状态、错误提示、成功确认
4. **代码复用**: Hook 封装业务逻辑，组件只负责 UI
5. **类型安全**: 完整的 TypeScript 类型定义
