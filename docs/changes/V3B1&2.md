### 1. notifications.service.ts — 添加 createNotification 方法

新增 `createNotification` 方法：

```typescript
async createNotification(params: {
  userId: string;
  type: 'COMMENT' | 'LIKE' | 'FOLLOW' | 'SYSTEM';
  message: string;
  actorId?: string;
  articleId?: string;
  link?: string;
}) {
  const { userId, type, message, actorId, articleId, link } = params;

  // Don't notify yourself
  if (actorId === userId) {
    return { success: true };
  }

  await this.prisma.notification.create({
    data: {
      userId,
      type,
      message,
      actorId,
      articleId,
      link,
    },
  });

  return { success: true };
}
```

**设计说明**：

- 自动跳过给自己发通知的场景（`actorId === userId`）
- 支持四种通知类型：COMMENT、LIKE、FOLLOW、SYSTEM
- 通过 `actorId` 关联触发者，用于显示头像等信息
- 通过 `articleId` 关联文章，用于跳转
- 通过 `link` 直接指定跳转链接

## 通知触发场景

### LIKE 通知

```
用户 A 点赞用户 B 的文章
  ↓
ArticlesService.like() 执行
  ↓
调用 notificationsService.createNotification()
  ↓
创建 LIKE 类型通知，userId 为 B（文章作者）
  ↓
B 在通知列表看到：A 赞了你的文章
```

### FOLLOW 通知

```
用户 A 关注用户 B
  ↓
UsersService.follow() 执行
  ↓
调用 notificationsService.createNotification()
  ↓
创建 FOLLOW 类型通知，userId 为 B（被关注者）
  ↓
B 在通知列表看到：A 关注了你
```
