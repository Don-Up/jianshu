### 2. UploadModule (`apps/api/src/upload/upload.module.ts`)

```typescript
@Module({
  imports: [
    MulterModule.register({
      limits: {
        fileSize: 5 * 1024 * 1024, // 5MB
      },
    }),
  ],
  controllers: [UploadController],
  providers: [
    UploadService,
    LocalStorageProvider,
    CloudinaryStorageProvider,
    {
      provide: 'STORAGE_PROVIDER',
      useFactory: () =>
        process.env.CLOUDINARY_URL
          ? new CloudinaryStorageProvider()
          : new LocalStorageProvider(),
    },
  ],
  exports: [UploadService],
})
export class UploadModule {}
```

**设计要点**:
- 使用 `STORAGE_PROVIDER` token 实现依赖注入
- 根据环境变量 `CLOUDINARY_URL` 是否配置自动选择存储提供者
- 同时注册 `LocalStorageProvider` 和 `CloudinaryStorageProvider`，但仅在工厂函数中实例化一个

### 3. UploadController (`apps/api/src/upload/upload.controller.ts`)

```typescript
@ApiTags('upload')
@Controller('upload')
export class UploadController {
  @Post()
  @UseGuards(JwtAuthGuard)
  @ApiBearerAuth()
  @UseInterceptors(
    FileInterceptor('file', {
      limits: { fileSize: 5 * 1024 * 1024 }, // 5MB
      fileFilter: (req, file, callback) => {
        const allowedMimes = ['image/jpeg', 'image/png', 'image/webp'];
        if (!allowedMimes.includes(file.mimetype)) {
          return callback(new BadRequestException('Only JPEG, PNG, and WebP images are allowed'), false);
        }
        callback(null, true);
      },
    }),
  )
  @ApiConsumes('multipart/form-data')
  @ApiOperation({ summary: 'Upload an image' })
  async uploadFile(@UploadedFile() file: Express.Multer.File) {
    const result = await this.uploadService.uploadFile(file);
    return {
      success: true,
      data: {
        url: result.url,
        mimetype: result.mimetype,
        size: result.size,
      },
    };
  }
}
```

**特性**:
- 路由: `POST /api/upload`
- 需要 JWT 认证 (`@UseGuards(JwtAuthGuard)`)
- 文件字段名: `file`
- 支持格式: `image/jpeg`, `image/png`, `image/webp`
- 文件大小限制: 5MB

### 4. StorageProvider 接口 (`apps/api/src/upload/providers/storage-provider.interface.ts`)

```typescript
export interface UploadResult {
  url: string;
  filename: string;
  mimetype: string;
  size: number;
}

export interface StorageProvider {
  upload(file: Express.Multer.File): Promise<UploadResult>;
  delete?(url: string): Promise<void>;
}
```

**设计要点**:
- 抽象存储逻辑，便于未来扩展 S3、GCS 等其他存储服务
- 定义 `UploadResult` 统一返回格式
- `delete` 方法为可选实现

### 5. LocalStorageProvider (`apps/api/src/upload/providers/local-storage.provider.ts`)

```typescript
@Injectable()
export class LocalStorageProvider implements StorageProvider {
  async upload(file: Express.Multer.File): Promise<UploadResult> {
    const uploadDir = join(process.cwd(), this.dest);
    if (!existsSync(uploadDir)) {
      await mkdir(uploadDir, { recursive: true });
    }
    const filename = `${Date.now()}-${file.originalname}`;
    const filepath = join(uploadDir, filename);
    await writeFile(filepath, file.buffer);
    return {
      url: `/uploads/${filename}`,
      filename,
      mimetype: file.mimetype,
      size: file.size,
    };
  }
}
```

**特性**:
- 文件存储到 `./uploads/` 目录
- 文件名格式: `{timestamp}-{originalname}`
- 返回相对路径 `/uploads/{filename}`

### 6. CloudinaryStorageProvider (`apps/api/src/upload/providers/cloudinary-storage.provider.ts`)

```typescript
@Injectable()
export class CloudinaryStorageProvider implements StorageProvider {
  constructor() {
    cloudinary.config({ cloudinary_url: process.env.CLOUDINARY_URL });
  }

  async upload(file: Express.Multer.File): Promise<UploadResult> {
    return new Promise((resolve, reject) => {
      const folder = process.env.CLOUDINARY_FOLDER || 'jianshu/uploads';
      cloudinary.uploader.upload_stream(
        { folder, resource_type: 'auto', public_id: `${Date.now()}-${file.originalname}` },
        (error, result) => {
          if (error || !result) {
            reject(error || new Error('Cloudinary upload failed'));
            return;
          }
          resolve({
            url: result.secure_url,
            filename: result.public_id,
            mimetype: file.mimetype,
            size: file.size,
          });
        },
      ).end(file.buffer);
    });
  }
}
```

## API 文档

### POST /api/upload

**描述**: 上传图片

**认证**: Bearer Token (JWT)

**Content-Type**: `multipart/form-data`

**请求体**:
| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| file | File | 是 | 图片文件 |

**支持格式**: JPEG, PNG, WebP

**文件大小限制**: 5MB

**响应示例**:
```json
{
  "success": true,
  "data": {
    "url": "/uploads/1750245907123-example.jpg",
    "mimetype": "image/jpeg",
    "size": 123456
  }
}
```

**错误响应**:
- `400 Bad Request`: 文件类型不支持或文件大小超出限制
- `401 Unauthorized`: 未提供或无效的 JWT token

## 生产环境部署

### 使用 Cloudinary

1. 注册 Cloudinary 账号
2. 在 Cloudinary Dashboard 获取 `CLOUDINARY_URL`
3. 在 `.env` 中设置:
   ```env
   CLOUDINARY_URL=cloudinary://api_key:api_secret@cloud_name
   CLOUDINARY_FOLDER=jianshu/uploads
   ```
4. 重启服务

**注意**: 配置 Cloudinary 后，本地静态文件服务将自动禁用。

## 设计亮点

1. **存储抽象**: 通过 `StorageProvider` 接口解耦存储逻辑，便于扩展
2. **自动降级**: 无 Cloudinary 配置时自动使用本地存储，开发友好
3. **双重限制**: 在 MulterModule 和 FileInterceptor 两处设置文件大小限制
4. **格式校验**: 在 FileInterceptor 中校验 MIME 类型
5. **统一响应格式**: 与其他 API 保持一致的 `{ success, data }` 格式
