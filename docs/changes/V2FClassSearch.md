# SearchBar 和 SearchInput 文本类图

## SearchBar — 搜索表单

```
┌─────────────────────────────────────┐
│           SearchBar 组件              │
├─────────────────────────────────────┤
│  状态                                │
│  └── query: string                   │
├─────────────────────────────────────┤
│  属性                                │
│  └── (无自有属性)                     │
├─────────────────────────────────────┤
│  方法                                │
│  ├── handleSubmit(e)                 │
│  └── handleKeyDown(e)                │
└─────────────────────────────────────┘
          │
          │ 使用
          ▼ (useRouter)
    ┌─────────────┐
    │  跳转逻辑    │
    └─────────────┘
          │
          ▼
    /search?q={query} 
```

**SearchBar 特点**:

- 简单表单组件，只有 query 状态
- handleSubmit 和 handleKeyDown 都调用同一个提交逻辑
- 提交后跳转到搜索结果页，URL 包含搜索参数
- 不直接调用 API，不显示结果

---

## SearchInput — 实时搜索下拉

```
┌─────────────────────────────────────────────┐
│              SearchInput 组件                 │
├─────────────────────────────────────────────┤
│  状态                                      │
│  ├── query: string                         │
│  ├── results: ArticleWithAuthor[]          │
│  ├── isLoading: boolean                     │
│  └── isOpen: boolean                       │
├─────────────────────────────────────────────┤
│  引用                                      │
│  ├── inputRef: RefObject                   │
│  └── dropdownRef: RefObject                 │
├─────────────────────────────────────────────┤
│  方法                                      │
│  ├── searchArticles(query)                  │
│  └── handleResultClick(slug)               │
└─────────────────────────────────────────────┘
          │
          │ 调用
          ▼
    ┌─────────────────┐
    │  articleApi     │
    │  .list()        │
    └─────────────────┘
          │
          ▼
    ┌─────────────────┐
    │  返回结果       │
    │  results        │
    └─────────────────┘
          │
          ▼
    ┌─────────────────────────────────────┐
    │           下拉结果列表                 │
    │  ┌─────────────────────────────┐     │
    │  │ [头像] 标题              ● │     │
    │  │        作者 · 日期         │     │
    │  └─────────────────────────────┘     │
    │  ┌─────────────────────────────┐     │
    │  │ [头像] 标题              ● │     │
    │  │        作者 · 日期         │     │
    │  └─────────────────────────────┘     │
    └─────────────────────────────────────┘
```

**SearchInput 特点**:

- 复杂组件，有多个状态
- 调用 articleApi.list 实时搜索
- 300ms 防抖避免频繁请求
- 点击外部关闭下拉
- 点击结果跳转到文章页

---

## 两者对比

```
SearchBar                              SearchInput
┌─────────────────┐    vs    ┌─────────────────────┐
│ 简单表单         │         │ 复杂交互             │
│ 提交跳转         │         │ 实时下拉             │
│ URL 可分享       │         │ 无 URL 参数          │
└─────────────────┘         └─────────────────────┘

SearchBar 流程:

  输入 ──► 回车 ──► 跳转 ──► SearchPage
                            │
                            ▼
                      /search?q=关键词

SearchInput 流程:

  输入 ──► 300ms ──► API ──► 下拉 ──► 点击结果 ──► 文章页
                              │
                              ▼
                         显示结果列表
```

## SearchInput 内部状态变化

```
初始状态:
  query = ""
  results = []
  isLoading = false
  isOpen = false

用户输入 "测试":
  query = "测试"
  (300ms 后) isLoading = true
  调用 articleApi.list({ search: "测试" })
  API 返回后:
    results = [文章1, 文章2, 文章3]
    isLoading = false
    isOpen = true

用户点击结果 "文章1":
  调用 router.push("/article/article-1-slug")
  query = ""
  results = []
  isOpen = false

用户点击外部:
  isOpen = false
  (results 保留，用户再次输入时会用同样的 results)
```
