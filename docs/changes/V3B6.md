## 概述

## 详细改动

### 1. main.ts 启用版本控制

```typescript
import { VersioningType } from '@nestjs/common';

// 在 bootstrap() 函数中
app.enableVersioning({
  type: VersioningType.URI,  // URL-based versioning: /api/v1, /api/v2
  defaultVersion: '1',       // 默认版本
  prefix: 'v',               // 前缀: v1, v2
});

app.setGlobalPrefix('api');
```

**配置说明**:
- `type: VersioningType.URI` - 通过 URL 路径进行版本控制
- `defaultVersion: '1'` - 未指定版本时默认使用 v1
- `prefix: 'v'` - 版本号前缀，如 `v1`、`v2`

### 2. 控制器版本装饰器

所有控制器从 `@Controller('xxx')` 改为 `@Controller({ path: 'xxx', version: '1' })`：

```typescript
// 之前
@Controller('articles')
export class ArticlesController {}

// 之后
@Controller({ path: 'articles', version: '1' })
export class ArticlesController {}
```

### 3. Swagger 版本更新

```typescript
const config = new DocumentBuilder()
  .setTitle('Jianshu API')
  .setDescription('简书博客平台 API')
  .setVersion('1.0')  // 从 '2.0' 改为 '1.0'
  .addBearerAuth()
  .build();
```

## URL 结构

### 启用版本控制后

```
GET /api/v1/articles         # 文章列表 (v1)
GET /api/v1/auth/login       # 登录 (v1)
GET /api/v1/users/:username   # 用户信息 (v1)
POST /api/v1/upload          # 上传文件 (v1)

# 未来 v2 API
GET /api/v2/articles         # 文章列表 (v2)
```

### 向后兼容

由于设置了 `defaultVersion: '1'`，旧的无版本请求会自动重定向到 v1：

```
GET /api/articles   -> 自动路由到 /api/v1/articles
GET /api/auth/login  -> 自动路由到 /api/v1/auth/login
```

## 未来扩展 v2 API

当需要发布 v2 API 时，只需：

1. 创建新的控制器或方法，添加 `version: '2'`

```typescript
@Controller({ path: 'articles', version: '2' })
export class ArticlesV2Controller {}
```

2. 在同一路径上可以同时存在多个版本：

```typescript
// v1 控制器
@Controller({ path: 'articles', version: '1' })
export class ArticlesController {}

// v2 控制器
@Controller({ path: 'articles', version: '2' })
export class ArticlesV2Controller {}
```

3. 客户端通过 URL 选择版本：

```bash
curl http://localhost:4000/api/v1/articles  # v1
curl http://localhost:4000/api/v2/articles  # v2
```

## Breaking Changes 策略

当需要发布 breaking changes 时：

1. **不要删除旧的 v1 端点**，而是：
   - 在旧端点上添加 `@Deprecated()` 装饰器
   - 在响应头或响应体中添加 deprecation 警告
   - 在文档中标注废弃时间线和移除计划

2. **创建新的 v2 端点**：
   - 保持清晰的版本变更说明
   - 提供从 v1 到 v2 的迁移指南

3. **监控使用情况**：
   - 通过日志分析 v1 和 v2 的使用比例
   - 在适当的时机移除旧版本

## 设计亮点

1. **NestJS 内置支持**: 使用官方提供的版本控制机制，稳定可靠
2. **URL 版本控制**: 通过 URI 路径直观区分版本，如 `/api/v1/xxx`
3. **默认版本兼容**: 设置默认版本为 '1'，使现有客户端无需修改
4. **灵活扩展**: 支持同一路径的多个版本共存
5. **Swagger 集成**: 自动在 API 文档中显示版本信息
