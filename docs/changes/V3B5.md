### 2. WsJwtGuard (`apps/api/src/auth/guards/ws-jwt.guard.ts`)

```typescript
@Injectable()
export class WsJwtGuard implements CanActivate {
  constructor(private jwtService: JwtService) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const client: Socket = context.switchToWs().getClient();
    const token = this.extractToken(client);

    if (!token) {
      throw new WsException('Unauthorized: No token provided');
    }

    try {
      const payload = await this.jwtService.verifyAsync<JwtPayload>(token, {
        secret: process.env.JWT_SECRET || 'default-secret',
      });

      // Attach user to socket data for later use
      client.data.user = {
        id: payload.sub,
        email: payload.email,
      };

      return true;
    } catch (error) {
      throw new WsException('Unauthorized: Invalid token');
    }
  }

  private extractToken(client: Socket): string | undefined {
    // Try authorization header first (standard Bearer token)
    const authHeader = client.handshake.headers.authorization;
    if (authHeader?.startsWith('Bearer ')) {
      return authHeader.substring(7);
    }

    // Fallback to token query parameter (for WebSocket connections)
    return client.handshake.query.token as string;
  }
}
```

**设计要点**:
- 实现 `CanActivate` 接口，与 HTTP 守卫模式一致
- 从 `handshake.headers.authorization` 提取 Bearer Token
- 也支持 `handshake.query.token` 作为备选
- 验证成功后，将用户信息挂载到 `client.data.user`
- 使用 `WsException` 而非普通异常，确保 WebSocket 错误正确传递

### 3. NotificationsGateway (`apps/api/src/gateway/notifications.gateway.ts`)

```typescript
@WebSocketGateway({
  namespace: '/notifications',
  cors: {
    origin: process.env.CORS_ORIGINS?.split(',') || ['http://localhost:3000'],
    credentials: true,
  },
})
export class NotificationsGateway
  implements OnGatewayConnection, OnGatewayDisconnect
{
  @WebSocketServer()
  server!: Server;

  @SubscribeMessage('join')
  @UseGuards(WsJwtGuard)
  handleJoin(
    @MessageBody() data: any,
    @ConnectedSocket() client: AuthenticatedSocket,
  ) {
    const userId = client.data.user?.id;
    if (!userId) {
      return { event: 'error', data: { message: 'User not authenticated' } };
    }

    client.join(`user:${userId}`);
    return { event: 'joined', data: { userId } };
  }

  @SubscribeMessage('leave')
  @UseGuards(WsJwtGuard)
  handleLeave(
    @MessageBody() data: any,
    @ConnectedSocket() client: AuthenticatedSocket,
  ) {
    const userId = client.data.user?.id;
    if (userId) {
      client.leave(`user:${userId}`);
    }
    return { event: 'left', data: { userId } };
  }

  // Called by NotificationsService to push notifications to users
  notifyUser(userId: string, notification: any) {
    this.server.to(`user:${userId}`).emit('notification', notification);
  }

  // Broadcast to all connected clients (for system notifications)
  broadcastNotification(notification: any) {
    this.server.emit('notification', notification);
  }
}
```

**设计要点**:
- 使用 `/notifications` 命名空间，与 HTTP API 分离
- 实现 `OnGatewayConnection` 和 `OnGatewayDisconnect` 生命周期接口
- 用户加入房间 `user:{userId}`，实现多设备同步
- `notifyUser()` 方法供 `NotificationsService` 调用
- `broadcastNotification()` 用于系统广播通知

## 循环依赖处理

`NotificationsService` 和 `NotificationsGateway` 之间存在循环依赖：

```
NotificationsService → NotificationsGateway
         ↑
    GatewayModule
```

**解决方案**: 使用 `forwardRef()` 延迟解析：

```typescript
// notifications.service.ts
@Inject(forwardRef(() => NotificationsGateway))
private notificationsGateway: NotificationsGateway;
```

## WebSocket 事件

### 客户端发送

| 事件 | 说明 | 认证 |
|------|------|------|
| `join` | 加入用户通知房间 | 需要 |
| `leave` | 离开通知房间 | 需要 |

### 服务端发送

| 事件 | 说明 | 触发条件 |
|------|------|---------|
| `joined` | 加入成功确认 | 用户调用 `join` 后 |
| `left` | 离开成功确认 | 用户调用 `leave` 后 |
| `notification` | 新通知 | `createNotification()` 被调用 |

## 客户端集成示例

### 方式一：使用 Socket.io 客户端

```typescript
import { io, Socket } from 'socket.io-client';

const socket: Socket = io('http://localhost:4000/notifications', {
  query: { token: 'your_jwt_token_here' },
  transports: ['websocket'],
});

socket.on('connect', () => {
  console.log('Connected to notifications gateway');
  socket.emit('join'); // Join user room
});

socket.on('joined', (data) => {
  console.log('Joined room:', data.userId);
});

socket.on('notification', (notification) => {
  console.log('New notification:', notification);
  // Update UI, show toast, etc.
});

socket.on('disconnect', () => {
  console.log('Disconnected from notifications gateway');
});
```

### 方式二：原生 WebSocket

```javascript
const ws = new WebSocket('ws://localhost:4000/notifications?token=your_jwt_token');

ws.onopen = () => {
  ws.send(JSON.stringify({ event: 'join', data: {} }));
};

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  if (message.event === 'notification') {
    console.log('New notification:', message.data);
  }
};
```

## 生产环境注意事项

1. **CORS 配置**: 确保 `CORS_ORIGINS` 环境变量包含前端域名
2. **WebSocket 升级超时**: 默认 10 秒，如需调整：
   ```typescript
   @WebSocketGateway({
     namespace: '/notifications',
     cors: { origin: '*', credentials: true },
     pingTimeout: 60000,
     pingInterval: 25000,
   })
   ```
3. **负载均衡**: WebSocket 需要 sticky session 支持，确保使用 `socket.io-redis` 适配器
4. **认证安全**: 生产环境建议使用 `wss://` 和 HTTPS
