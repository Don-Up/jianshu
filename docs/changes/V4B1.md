# Task 1: Prisma Schema - Comment & CommentLike Models

## 第一部分: 概述

为评论系统扩展数据模型支持无限嵌套回复和评论点赞功能。在 `Comment` 模型中添加 `parentId` 自引用实现无限嵌套结构，新增 `CommentLike` 模型实现评论点赞（防止重复点赞的 unique 约束），并建立 User 与 CommentLike 的关联关系。

## 第三部分: 详细改动说明

### schema.prisma[修改](apps/api/prisma/schema.prisma)

**1. User 模型添加 commentLikes 关系**

```prisma
model User {
  // ... 其他字段 ...
  articles     Article[]
  comments     Comment[]
  commentLikes CommentLike[]  // 新增
  likes        Like[]
  bookmarks    Bookmark[]
  // ...
}
```

**说明：** User 可以通过 `commentLikes` 访问其点赞过的评论。

---

**2. Comment 模型扩展 - 支持无限嵌套**

```prisma
model Comment {
  id        String   @id @default(cuid())
  content   String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  authorId  String
  author    User    @relation(fields: [authorId], references: [id], onDelete: Cascade)

  articleId String
  article   Article @relation(fields: [articleId], references: [id], onDelete: Cascade)

  parentId  String?                           // 新增：父评论ID
  parent    Comment?  @relation("CommentReplies", fields: [parentId], references: [id], onDelete: Cascade)
  replies   Comment[] @relation("CommentReplies")  // 新增：子评论

  likes     CommentLike[]                     // 新增：关联点赞

  @@index([articleId])
  @@index([authorId])
  @@index([parentId])                         // 新增：加速嵌套查询
  @@map("comments")
}
```

**说明：**
- `parentId` 为 null 表示顶层评论，非 null 表示回复
- `replies` 自引用实现无限嵌套结构
- 添加 `@@index([parentId])` 加速嵌套评论查询

---

**3. CommentLike 模型[新增]**

```prisma
model CommentLike {
  id        String   @id @default(cuid())
  createdAt DateTime @default(now())

  commentId String
  comment   Comment @relation(fields: [commentId], references: [id], onDelete: Cascade)

  userId    String
  user      User    @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([commentId, userId])  // 防止重复点赞
  @@index([commentId])
  @@index([userId])
  @@map("comment_likes")
}
```

**说明：**
- `@@unique([commentId, userId])` 确保同一用户不能重复点赞同一评论
-  Cascade 删除：删除评论或用户时自动删除对应点赞记录

## 其它

### 设计亮点

- **无限嵌套**：通过 `parentId` 自引用实现，可支持任意深度的回复嵌套
- **防止重复点赞**：`@@unique([commentId, userId])` 约束在数据库层面保证数据一致性
- **级联删除**：Cascade 配置确保删除评论或用户时自动清理关联数据
