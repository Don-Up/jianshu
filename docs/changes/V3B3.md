## 1. Redis Service

**文件**: `apps/api/src/redis/redis.service.ts`

Redis 服务类封装了 ioredis 客户端：

```typescript
@Injectable()
export class RedisService implements OnModuleInit, OnModuleDestroy {
  private client!: Redis;

  async onModuleInit() {
    this.client = new Redis({
      host: process.env.REDIS_HOST || 'localhost',
      port: parseInt(process.env.REDIS_PORT || '6379'),
    });
  }
}
```

**关键方法**：

| 方法 | 功能 |
|------|------|
| `get(key)` | 获取缓存值 |
| `set(key, value, ttl?)` | 设置缓存，支持过期时间 |
| `del(key)` | 删除指定 key |
| `delPattern(pattern)` | 删除匹配 pattern 的所有 key，如 `articles:*` |

**设计说明**：
- 使用 `!` 非空断言因为 client 在 `onModuleInit` 中初始化
- `delPattern` 用于批量清除缓存，如修改文章时清除所有文章列表缓存

## 2. Redis Module

**文件**: `apps/api/src/redis/redis.module.ts`

```typescript
@Global()
@Module({
  providers: [RedisService],
  exports: [RedisService],
})
export class RedisModule {}
```

**设计说明**：
- `@Global()` 装饰器使 RedisService 全局可用
- 所有模块无需导入即可注入 RedisService
- 类似 PrismaModule 的设计模式

### 4.2 findAll 缓存读取

```typescript
async findAll(query: QueryArticleDto, userId?: string) {
  const cacheKey = `articles:list:${JSON.stringify({ page, limit, authorId, tag, search })}:${userId || 'anonymous'}`;

  // 缓存命中直接返回
  const cached = await this.redis.get(cacheKey);
  if (cached) {
    return JSON.parse(cached);
  }

  // ... 数据库查询 ...

  const result = { success: true, data: { items, total, ... } };

  // 缓存 5 分钟
  await this.redis.set(cacheKey, JSON.stringify(result), 300);

  return result;
}
```

**缓存键设计**：
```
articles:list:{"page":1,"limit":20,"authorId":null,"tag":null,"search":"关键词"}:anonymous
articles:list:{"page":1,"limit":20,"authorId":"user-123",...}:user-123
```

### 4.3 缓存失效（create/update/delete/like）

```typescript
// create 后
await this.redis.delPattern('articles:list:*');

// update 后
await this.redis.delPattern('articles:list:*');

// delete 后
await this.redis.delPattern('articles:list:*');

// like 后（取消点赞或新增点赞都清除缓存）
await this.redis.delPattern('articles:list:*');
```

**设计说明**：
- 使用 `delPattern('articles:list:*')` 清除所有文章列表缓存
- 不清除单个缓存因为查询键太复杂（包含分页、筛选条件等）
- 简单粗暴的清除策略，牺牲一些缓存命中率换取代码简单

## 5. 缓存策略总结

| 数据类型 | 缓存键 | TTL | 失效方式 |
|---------|---------|-----|---------|
| 文章列表 | `articles:list:*` | 5分钟 | create/update/delete/like |

**未来扩展**：

| 数据类型 | 缓存键 | TTL | 失效方式 |
|---------|---------|-----|---------|
| 用户资料 | `user:{username}` | 10分钟 | profile update |
| 通知计数 | `notifications:unread:{userId}` | 1分钟 | markAsRead/markAllAsRead |
