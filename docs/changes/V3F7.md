## 第三部分: 详细改动说明

### query-keys.ts[新增](apps/web/src/lib/query-keys.ts)

```typescript
export const queryKeys = {
  auth: ['auth'] as const,
  authMe: () => [...queryKeys.auth, 'me'] as const,

  articles: ['articles'] as const,
  article: (slug: string) => [...queryKeys.articles, slug] as const,

  users: ['users'] as const,
  user: (username: string) => [...queryKeys.users, username] as const,
  userArticles: (username: string) => [...queryKeys.users, username, 'articles'] as const,

  notifications: ['notifications'] as const,
};
```

**说明：**
- 提供集中化的 Query Key 管理，避免字符串硬编码
- 使用 `as const` 保证类型安全
- 函数式写法如 `authMe()` 方便扩展

### use-article.ts[修改](apps/web/src/hooks/use-article.ts)

**主要改动：**
- `useArticle` 重构为使用 `useQuery` 获取文章数据
- 新增 `useArticleList` 用于文章列表查询
- 新增 `useCreateArticle` 用于创建文章
- 新增 `useUpdateArticle` 用于更新文章
- Like 操作用 `setQueryData` 直接更新缓存

```typescript
export function useArticle(slug: string) {
  const queryClient = useQueryClient();

  const query = useQuery({
    queryKey: queryKeys.article(slug),
    queryFn: () => articleApi.getBySlug(slug),
    enabled: !!slug,
  });

  const likeMutation = useMutation({
    mutationFn: () => articleApi.like(slug),
    onSuccess: (res) => {
      if (res.success && res.data) {
        queryClient.setQueryData(
          queryKeys.article(slug),
          (old) => {
            if (old?.data) {
              return {
                ...old,
                data: {
                  ...old.data,
                  likeCount: res.data!.likeCount,
                  isLiked: !old.data.isLiked,
                },
              };
            }
            return old;
          }
        );
      }
    },
  });

  return {
    article: query.data?.data ?? null,
    isLoading: query.isLoading,
    error: query.error?.message ?? null,
    likeArticle: likeMutation.mutate,
    isLiking: likeMutation.isPending,
    refetch: query.refetch,
  };
}
```

## 其它

### 设计亮点

- **统一 Key 管理**：`queryKeys` 工厂函数集中管理所有查询键，避免散落在各处
- **缓存更新策略**：Like 等操作使用 `setQueryData` 直接更新缓存，用户无感知
- **失效传播**：创建/更新文章后正确失效相关缓存，确保数据一致
